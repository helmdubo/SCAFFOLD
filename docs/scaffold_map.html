3 · Решение stitch vs split — двухуровневая рамка

Уровень A — жёсткий, геометрический Layer 2/3 + solve

  • Развёртываемость: |2π − Σуглов| у вершин, становящихся внутренними. Сгиб стен = 0 → шить можно; обод капа = π/2 → запрет.
  • T-стык: конец шва с оставшимся разрезом не становится внутренним → исключён из гейта.
  • SEAM_SELF (труба): шить = отменить разрез → всегда split.

Уровень B — мягкий, семантический Layer 4/5, отложено

  • Когда дисторсия ≈ 0 (стены куба) — геометрия не может решить.
  • Решают: трим-политика, ориентация текселя, упаковка, грамматика.
  • Поздние роли (WALL/FLOOR) могут вернуться только здесь как runtime-метки.

5 · Anti-CFTUV guards (действуют на каждый срез)

  • 1 · Спайк приговорён: после G4 — контракты и чистая реализация, спайк в архив; импорт из него запрещён всем.
  • 2 · Нет хранимых ролей: оси — вычисляемые проекции families, пересчёт каждый прогон.
  • 3 · Solve не пишет назад: «слой L3 должен был сказать X» → consumption report → отдельный L3-срез.
  • 4 · Скелет = всё выделение, не патч: row/col-графы поверх всех островов (mesh/shell-уровень).
  • 5 · Семантические правила → L4: в спайке только LEVEL_B_PLACEHOLDER.
  • 6 · Размер: файл ≤ ~600 строк, спайк ≤ ~3000; монолит = смерть CFTUV, не повторяем.
